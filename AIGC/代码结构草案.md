# QuickLauncher 代码结构草案

```text
QuickLauncher/
├─ Models/
│  ├─ HotkeyGesture.cs
│  └─ UserSettings.cs
├─ Services/
│  ├─ HotkeyService.cs
│  ├─ SettingsService.cs
│  ├─ StartupService.cs
│  └─ TrayService.cs
├─ Infrastructure/
│  └─ NativeMethods.cs
├─ MainWindow.xaml
├─ MainWindow.xaml.cs
├─ SettingsWindow.xaml
├─ SettingsWindow.xaml.cs
├─ App.xaml
└─ App.xaml.cs
```

## 结构原则

- 第一阶段只放真正会参与运行的文件
- 不为尚未实现的索引、搜索模块提前铺过多空目录
- 等第二阶段开始再补 `AppEntry`、`SearchResult`、`AppIndexService`、`SearchService`

## 第一阶段职责

- `App.xaml.cs`
  - 应用入口
  - 主窗口与设置窗口生命周期
  - 托盘和热键协作
- `MainWindow`
  - 搜索浮层界面
  - 响应关闭、隐藏、聚焦
- `SettingsWindow`
  - 编辑基础配置
- `SettingsService`
  - 读写配置
- `StartupService`
  - 处理开机启动
- `HotkeyService`
  - 注册和释放全局热键
- `TrayService`
  - 托盘图标与菜单

## 第二阶段预留

- `Models/AppEntry.cs`
- `Models/SearchResult.cs`
- `Services/AppDiscoveryService.cs`
- `Services/AppIndexService.cs`
- `Services/SearchService.cs`
- `Services/LaunchService.cs`
